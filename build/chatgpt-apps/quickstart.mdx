### Prerequisites

Before you begin, make sure you have:

- **Node.js 18+** installed (or Python 3.10+ for Python projects)
- **Basic React knowledge** - Familiarity with React components and hooks
- **Smithery API key** - Get one free at [smithery.ai](https://smithery.ai)
- **MCP concepts** - Understanding of [MCP basics](/concepts/what_is_mcp)

### 1. Create Your App

Use the ChatGPT app template to scaffold a complete project:

```bash
npm create smithery@latest my-app -- --template chatgpt-app
cd my-app
```

### 2. Start Development

```bash
npm run dev
```

This starts the development server with:
- Hot reload for server and widget code
- Automatic rebuilds on file changes
- Local testing environment

Now try prompting: "Say hello world!" and watch your widget appear!

### 3. Project Structure

Your new project has this structure:

```
my-app/
├── app/
│   ├── server/
│   │   └── index.ts           # MCP server code
│   ├── shared/
│   │   └── types.ts           # Shared types (best practice)
│   └── web/
│       └── src/
│           └── widget.tsx     # React widget code
├── smithery.yaml              # Configuration
└── package.json
```

<Tip>
**Best Practice: Shared Types**

Use the `app/shared/` folder for TypeScript interfaces that are used by both server and client code. This gives you end-to-end type safety from tool response to React component.
</Tip>

## Understanding the Template

The template includes a complete working example showing the basic contract:

**Server** (`app/server/index.ts`)
- Define tools using `widget.resource()` from `@smithery/sdk`
- Return responses with `structuredData` for your widget

**Client** (`app/web/src/widget.tsx`)
- React component that reads data via `useToolOutput()` hook
- Responds to theme changes with `useTheme()`

**Shared** (`app/shared/types.ts`)
- TypeScript interfaces used by both server and client
